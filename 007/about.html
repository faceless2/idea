Helpful fields

This makes four changes to the edit fields

  1. The prefix/suffix fields are limited to 10 characters, because that's what they're limited to in the database! I don't know the length limits on other fields, but they're all currently given a maxlength of 60 - if the limits are different for a particular field, they should be set to that limit. Without this, data is truncated when saving. This one really is a no-brainer.
  2. Text fields have leading/trailing whitespace trimmed when their values are changed. Only applies to single line fields, eg names, places - not the biography.
  3. Pressing the "Esc" key in any field resets it to the value it was on load and blurs the focus. I've often wanted this feature. Works on select fields too, could probably be extended to radio buttons if desired.
  4. When a Death Date/Place is set to "still living", the value of the field is reset to empty and the field is disabled. To me this is an obvious validation step before saving, but I appreciate there are probably many profiles that already have a date/place and yet are also "still living". It won't affect those at all - the value is only changed if the status is changed to "still living".

maxlength is best set on the server side when creating the pages. The other three are "change" or "keypress" listeners which are added to the appropriate fields. They can easily be copied directly from script.js or this functinality merged the methods in edit_person.js, they're only a few lines.
